# Security Policy

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you've discovered a security vulnerability in ziv, I'd really
appreciate you reaching out privately so it can be addressed
responsibly before any public disclosure.

You can contact me directly by emailing:

**ziv.ai.786@gmail.com**

To help me understand and resolve the issue as quickly as possible,
please include:

- A clear description of the vulnerability
- Steps to reproduce it
- The version of ziv you are using (`ziv --version`)
- Your OS and Python version

I will do my best to acknowledge your report within 48 hours and
aim to resolve confirmed vulnerabilities within 14 days.

## General Feedback

For general bugs, query issues, or feature requests, the easiest
way to reach me is through the built-in feedback command:

    ziv feedback

---

## Thank You

Security research is a genuinely valuable contribution to open
source software. I deeply appreciate the time, care, and effort
you put into responsibly disclosing vulnerabilities rather than
exploiting them. Every report helps make ziv safer and more
trustworthy for everyone who uses it. Thank you sincerely.
